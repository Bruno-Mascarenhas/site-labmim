Origem dos Dados — Modelo WRF

O que é o WRF?

O WRF (Weather Research and Forecasting) é um sistema de modelagem numérica atmosférica de mesoescala, desenvolvido colaborativamente pelo NCAR (National Center for Atmospheric Research), NOAA, AFWA e diversas universidades. É utilizado mundialmente tanto para pesquisa atmosférica quanto para previsão operacional do tempo, com aplicações que vão desde a simulação de furacões até a previsão de recursos energéticos renováveis.

O modelo resolve as equações primitivas de Euler (conservação de massa, momento e energia) em uma grade tridimensional, usando coordenadas verticais terrain-following (sigma). Processos físicos sub-grade (radiação, microfísica, turbulência, solo) são representados por parametrizações.

Parametrizações Físicas

  • Radiação ({{MODEL_RADIATION}}): Calcula os fluxos de radiação de onda curta e longa considerando interação com nuvens, aerossóis e gases atmosféricos.
  • Microfísica ({{MODEL_MICROPHYSICS}}): Simula os processos de formação e evolução de hidrometeoros (vapor, chuva, neve, graupel, gelo, gotículas de nuvem).
  • Camada Limite Planetária ({{MODEL_PBL}}): Parametriza o transporte turbulento de calor, umidade e momento na CLP.
  • Superfície Terrestre ({{MODEL_LAND_SURFACE}}): Modelo acoplado de solo e vegetação que calcula fluxos de calor sensível e latente na interface superfície-atmosfera.
  • Cumulus ({{MODEL_CUMULUS}}): Parametriza a convecção sub-grade nos domínios {{COARSE_DOMAIN_LABELS}}. Desativada em {{FINE_DOMAIN_LABELS}} (Δx < 4 km), onde a convecção é resolvida explicitamente.

Pipeline de Dados

Os arquivos brutos do WRF (NetCDF) são processados pelo pipeline Python {{DATA_PIPELINE_NAME}}:

  • Extração: Leitura das variáveis NetCDF via xarray / netCDF4.
  • Interpolação Vertical: Para variáveis eólicas, interpolação logarítmica dos níveis sigma para alturas de hub (50m, 100m, 150m).
  • Serialização: Conversão para GeoJSON (geometria da grade, 1 arquivo por domínio) e JSON (valores, 1 arquivo por domínio × variável × timestep). JSON compacto sem indentação para minimizar payload.
  • Vetores de Vento: Arquivo separado D0X_WIND_VECTORS_NNN.json com ângulos e magnitudes sub-amostrados (stride=4), usando U10/V10.

Limitações e Considerações

  • O erro das previsões numéricas cresce progressivamente com o horizonte temporal (especialmente após 48h).
  • A topografia complexa (como {{TERRAIN_EXAMPLE}}) pode introduzir viés local, especialmente em variáveis de superfície.
  • A resolução do modelo pode não capturar fenômenos convectivos isolados (< Δx) nos domínios {{COARSE_DOMAIN_LABELS}}.
  • A assimilação de dados observacionais locais (estações, radiossondas) melhoraria significativamente a acurácia, mas não está implementada nesta configuração.
  • Variáveis de fluxo de superfície (HFX, LH) são especialmente sensíveis à representação do uso do solo no modelo {{MODEL_LAND_SURFACE}}.
